1. Project Title & Overview

Project Title
Theodore Bunquin | Social Media Manager & Digital Strategist
Purpose
Portfolio website to present skills, experience, and service packages in social media management, photography, content creation, and web development.
Audience
Prospective clients, small businesses, student organizations, and recruiters evaluating digital marketing capability.

Objectives

  • Convey professional brand and core competencies with strong visual hierarchy.
  • Showcase experience via timeline and case-style portfolio cards.
  • Offer clear, tiered packages (SMM, Photography, Content, Web) to simplify decision-making.
  • Provide smooth navigation and responsive layout across devices.

Key Features

  • Animated hero with typing effect and CTAs; social links.
  • Stats counters and animated skill bars powered by IntersectionObserver.
  • Services grid with hover affordances and visual feedback.
  • Experience timeline and portfolio cards with staged entrance animations.
  • Packages page with tabs and package timelines.

Technology & Structure

Stack
  • HTML5 (semantic sections)
  • CSS3 with design tokens (CSS variables), grid, animations
  • Vanilla JavaScript for interactions and scroll observers
  • Font Awesome, Google Fonts (via CDN)

2. User Persona(s)

Persona A – Small Business Owner (Food & Beverage)

  • Goals: Increase local brand awareness and sales; establish consistent content cadence.
  • Behaviors: Searches for bundled SMM + content services; prefers clear pricing tiers.
  • Pain Points: Limited time; unsure how to measure performance; budget constraints.
  • How site helps: Service cards explain scope; packages provide tiered options; analytics offering builds trust.

Persona B – University Org Officer

  • Goals: Promote events; improve engagement; streamline content creation.
  • Behaviors: Skims examples; values case-style experience proof and quick contact.
  • Pain Points: Volunteer bandwidth; inconsistent visuals; little ad experience.
  • How site helps: Timeline and portfolio cards map to campus work; clear CTA paths.

Persona C – Recruiter / Hiring Manager

  • Goals: Validate skills and UX sense; gauge breadth (SMM, photo, content, web).
  • Behaviors: Scans hero, services, and experience quickly; notes polish and structure.
  • Pain Points: Portfolios lacking depth or clarity of role and outcomes.
  • How site helps: Concise achievements, skill bars, and professional presentation.

4. UI Design Principles Applied

  • Visual hierarchy: Gradient title treatments and larger type for hero (.name), clear section headers (.section-title, .section-tag).
  • Consistency & tokens: Color/spacing via CSS variables in :root; reusable components (cards, buttons, tags).
  • Affordance & feedback: Hover/active states on buttons, service cards, experience items; nav link active state updates on scroll.
  • Progressive disclosure: High-level services first; detailed packages on a separate page with tabs.
  • Motion for meaning: IntersectionObserver triggers counters/skill bars and staged card reveals—supports comprehension without distraction.
  • Responsive design: Grid layouts collapse at breakpoints; mobile nav with hamburger; timeline adapts to single column.
  • Readability: Sufficient contrast, ample spacing, and concise copy blocks; sectioned content for scanning.

Implementation References

  • Tokens & gradients: style.css (:root, --primary/--secondary, --dark-bg).
  • Components: Buttons (.btn), cards (.service-card, .experience-card), timeline (.timeline).
  • Interactions: script.js typing effect, counters, skill bars, nav highlighting, tab logic in packages page.
  • Responsive: Media queries at max-width: 768px adapting grid, timeline, nav.

5. Summary of Challenges & Learnings

Challenges

  • Balancing motion and performance: Multiple animated elements (counters, skill bars, cards) required careful use of IntersectionObserver and CSS transitions to avoid jank.
  • Mobile navigation: Ensuring the hamburger menu and nav state synced with scroll across sections without accidental overlaps.
  • Information architecture: Keeping the main page concise while offering detailed services—resolved by moving package details into a dedicated tabbed page.
  • Consistency at scale: Centralized variables and reusable components reduced style drift and improved maintainability.
  • Contact flow: Using mailto: avoids backend complexity but depends on user email client availability.

Key Learnings

  • Design tokens (CSS variables) accelerate iteration and enforce brand consistency.
  • IntersectionObserver is a lightweight, performant way to orchestrate scroll-based animations.
  • Clear CTAs and progressive disclosure improve decision pathways for diverse personas.
  • Modular page structure (index vs packages) clarifies the narrative and reduces cognitive load.
